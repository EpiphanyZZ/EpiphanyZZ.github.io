Git与Github常用操作方式

行命令进行操作

  • 主要步骤
    1. 终端cd至想要设置git的仓库文件夹
    2. 在终端输入git init
    3. add操作：使用git add “filename”进行add操作，或者直接用”git add .”将所有工作区的文件给add了
    4. commit操作：git commit -m “本次提交信息”
    5. 关联操作可在Github上创建仓库后根据提示进行
    6. push操作：git push 将本地仓库推送到远程仓库更新
  • 其他操作
    • git status 查找git状态
    • git remote -v 查看本仓库与远程仓库的信息
    • git pull 将远程仓库拉取至本地仓库进行更新

使用VSCode源代码管理进行操作

  • 主要步骤：
    1. 点开源代码管理的侧边栏
    2. 与Github上仓库关联或建立新仓库可根据VSCode自带提示进行
    3. 将工作区修改完毕的文件点击暂存更改(一个＋号)——相当于add操作
    4. 输入本次修改消息后点击提交——相当于commit操作
    5. 输入要上传到远程仓库的提交信息后点击同步更改——相当于push操作
  • 其他操作
    • 如果有在Github进行修改文件后保存commit，点击侧边栏的同步更改即可拉取文件进行更新——相当于pull操作
    • 也就是说同步操作中有↑代表push，有↓代表pull
